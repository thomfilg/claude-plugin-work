# Create Jira - Design Doc Evaluation

Referenced by: [create-jira.md](create-jira.md) Step 13.5

## When Design Docs Are Needed

Evaluate whether a design doc is needed based on task complexity. Simple tasks (bug fixes, adding tests to existing code) don't need design docs — the Jira task itself is sufficient documentation.

### Design Doc Triggers

```javascript
const DESIGN_DOC_TRIGGERS = {
  // Multiple domains involved suggests cross-cutting complexity
  agentCount: involvedAgents.length >= 3,

  // Multiple consensus iterations indicate non-trivial decisions
  iterations: iteration >= 2,

  // Keywords that suggest architectural or high-impact changes
  keywords: /\b(migrate|refactor|architecture|new system|breaking change|redesign|overhaul|replace|deprecate)\b/i.test(description),

  // Many out-of-scope items suggest complex scoping decisions worth documenting
  outOfScope: finalTask.outOfScope.length > 2
};

const triggerCount = Object.values(DESIGN_DOC_TRIGGERS).filter(Boolean).length;
const shouldOfferDesignDoc = triggerCount >= 2;
```

### When Design Docs Are NOT Needed

- Simple bug fixes
- Adding tests to existing functionality
- Minor UI tweaks
- Configuration changes
- Single-file changes with clear scope

## Offer Design Doc Generation

**If `shouldOfferDesignDoc` is true:**

```
AskUserQuestion:
  question: "This task has ${triggerCount} complexity indicators. Would you like to generate a design doc?"
  header: "Design Doc"
  options:
    - label: "Generate Design Doc"
      description: "Create design doc from Jira artifacts, then create Jira task"
    - label: "Skip Design Doc"
      description: "Just create the Jira task (recommended for simpler tasks)"
```

## Artifact Mapping

The create-jira artifacts map directly to design doc sections:

| create-jira Artifact | Maps to Design Doc Section |
|---------------------|---------------------------|
| `context.md` | Scope and Context |
| Agent pros/cons tables | Design (Trade-offs) |
| `consensus-log.md` | Alternatives Considered |
| Testing requirements | Testing and Observability |
| Out of scope items | Non-Objectives |
| Acceptance criteria | Objectives |

## Generate Design Doc

**If user selects "Generate Design Doc":**

```
Task(Explore):
  Generate a design doc from the Jira draft artifacts.

  **Source Directory:** ${DRAFT_DIR}

  **Template:** Use /create-design-doc template structure

  **Mapping:**
  1. Overview: Synthesize from final-task.md Summary
  2. Context and Scope: Extract from context.md
  3. Objectives: Convert acceptance criteria to measurable objectives
  4. Non-Objectives: Copy out-of-scope items from final-task.md
  5. Design: Merge technical approach sections + pros/cons from all agents
  6. Alternatives Considered: Extract from consensus-log.md disagreements
  7. Cross-cutting Concerns: Identify from agent consultation notes
  8. Testing and Observability: Merge testing plans + add rollback strategy
  9. Open Questions: Preserve from final-task.md

  **Output:** Save to ${DRAFT_DIR}/design-doc.md

  **Link Reference:** Add link to design doc in Jira task description
```

```markdown
**Tags:** design-doc, ${detected_task_types.join(', ')}

[From final-task.md Summary - 1-2 paragraphs]

[From context.md - current state, problem being solved]

## Design

### Solution Overview
[From final-task.md Technical Approach]

#### Backend Perspective
| Pros | Cons |
|------|------|

#### Frontend Perspective
| Pros | Cons |
|------|------|
[From frontend-v*.md]

#### DevOps Perspective
| Pros | Cons |
|------|------|
[From devops-v*.md]

## Alternatives Considered
[From consensus-log.md - extract disagreements and resolution decisions]

## Testing and Observability
### Testing Strategy
[From final-task.md Testing Plan]

[Add explicit rollback plan - triggers, procedures, data considerations]

${if /\b(migrate|migration|schema|data migration)\b/i.test(description)}

${endif}

```

## Save Design Doc Draft

```bash
# Save to drafts directory
Save design doc to: ${DRAFT_DIR}/design-doc.md

# Copy to tasks folder with descriptive name
TASK_DESIGN_DOC="tasks/design-docs/${SLUG}-design-doc.md"
mkdir -p tasks/design-docs

# Check if design doc already exists
if [ -f "$TASK_DESIGN_DOC" ]; then
  echo "⚠️ Design doc already exists: ${TASK_DESIGN_DOC}"
  # Offer to update or create versioned copy
  # AskUserQuestion: Update existing | Create new version (with timestamp)
  # If "Create new version": TASK_DESIGN_DOC="tasks/design-docs/${SLUG}-design-doc-$(date +%Y%m%d).md"
fi

cp "${DRAFT_DIR}/design-doc.md" "${TASK_DESIGN_DOC}"

echo "📄 Design doc draft saved to: ${TASK_DESIGN_DOC}"
echo "⏳ Wiki publishing will happen after your confirmation"
```
